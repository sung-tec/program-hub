# QA 보고서

## 구현 범위
- 학생 개인용 한국 초·중 수학 RPG 웹게임
- 로컬 익명 개인 코드 발급
- 학년, 학기, 단원, 난이도 변경
- 퀘스트 문제 풀이, 힌트, 해설, XP, 숙련도
- Phaser 기반 2.5D 장난감 피규어 캐릭터 렌더링
- 30,240개 모듈 조합형 아이템 카탈로그
- 머리, 귀걸이, 목걸이, 반지, 상의, 하의, 신발, 장갑, 왼손장비, 오른손장비 슬롯
- 도감 점수, 완성률, 부위별 진행률, 세트 진행률
- 무료 랜덤상자 확률표와 중복 조각 변환
- 결제, 광고, 유료 재화 미포함

## 성능 설계
- Phaser 렌더링은 60fps target, 저사양 모드에서는 45fps target
- 도감은 25,000개 이상 목록을 한 번에 DOM 렌더링하지 않고 가상 스크롤로 표시
- 아이템 이미지는 개별 파일 수만 장을 만들지 않고 색상, 소재, 슬롯 정보를 조합해 렌더링
- Vite 빌드는 Phaser와 React 청크를 분리해 초기 로딩 부담을 줄임

## 검수 기준
- `npm.cmd run verify`: 콘텐츠 상태, 아이템 목표 수, 상거래 금지어 확인
- `npm.cmd run build`: TypeScript와 Vite 정적 빌드 확인
- `npm.cmd run release`: 검수, 빌드, ZIP 생성

## 실행 결과
- `npm.cmd run verify`: 통과, 아이템 목표 수 30,240개 확인
- `npm.cmd run build`: 통과, `dist/` 생성 확인
- `npm.cmd run release`: 통과, `release.zip` 생성 확인
- 번들 구성: Phaser와 React 런타임을 별도 청크로 분리했다.
- `dist/START_GAME.ps1`: 로컬 정적 서버 실행 확인, `http://127.0.0.1:4174` 응답 확인
- 흰 화면 원인 수정: Phaser 씬 활성화 전 `isActive` 접근으로 앱 초기화가 중단되던 문제를 방어 처리했다.
- 수정 후 Chrome 헤드리스 렌더링 확인: `http://127.0.0.1:4173`, `http://127.0.0.1:4174`, `http://127.0.0.1:4175`에서 루트 UI와 캔버스 출력 확인
- `npm.cmd run qa:gameplay`: 통과. 실제 Chrome에서 초3 퀘스트 풀이, 무료 보상상자 지급, 도감 이동, 자동 장착, 도감 내 수동 장착, 중2 문항 출제를 확인했다.
- QA 표본: 초3 문항은 두 자리 수와 한 자리 수 곱셈, 중2 문항은 유리수의 소수 표현으로 출제됨을 확인했다.
- 저장 지속성 확인: 새로고침 후 개인 코드, XP, 레벨, 보유 아이템 수, 장착 슬롯, 수동 장착한 머리 아이템이 유지됨을 확인했다.

## 권리 검수
- 포함 문항은 런타임 생성형 자체 문항이다.
- 캐릭터와 아이템은 코드 기반 오리지널 조합 표현이다.
- 외부 이미지, 교과서 삽화, 문제집 문항, EBS 문항을 포함하지 않았다.
